9.defer loader！！！！！！！！
由于页面渲染在loader执行完成之后，loader里等待异步操作完成会阻塞渲染，
所以loader里只执行异步操作，不等待，删除await，返回值用defer（）包裹起来，此时传出去的值为Promise

loader内有多个请求时：const result = Promise.all([getCateListApi(), getArticleListApi(q)])

在loader没有父组件嵌套的组件里，要使用loaderData时：
    const loaderData = useLoaderData() as { result: Promise<BaseResponse<CateItem[]>> },
    此时loaderData为Promise
组件内    return (
        <Suspense fallback={'loading...'}>   //fallback内为：加载时渲染的东西
            <Await resolve={loaderData.result} errorElement='请求失败时展示的东西'>  //Await内为：加载完毕渲染的东西//xxx为resolve
               {(xxx)=>(return <>要渲染的东西</>)}
            </Await>
        </Suspense>
    )
!!!!!
在嵌套的子组件里（被<Suspense><Suspense/>包裹的组件），要使用异步的loaderData时：
    const loaderAsyncData = useAsyncValue() as { result: Promise<BaseResponse<CateItem[]>> },
    此时loaderData为Promise
组件内的return不用修改
